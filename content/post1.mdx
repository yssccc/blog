## ⚡ 프론트엔드 성능 최적화의 핵심

프론트엔드 개발을 하다 보면 “성능 최적화”는 언제나 숙제처럼 따라옵니다.  
빠르게 로드되고, 부드럽게 인터랙션하며, 불필요한 리렌더링이 없는 앱을 만드는 건 단순히 기술 스택의 문제가 아니라, **설계와 구조**의 문제이기도 합니다.

이번 글에서는 `Next.js` 프로젝트에서 실무적으로 적용 가능한 성능 최적화 방법을 단계별로 다뤄봅니다.

---

## 0 렌더링 전략: CSR, SSR, SSG, ISR

Next.js의 가장 강력한 특징은 다양한 렌더링 방식을 지원한다는 점입니다.

예를 들어, 블로그의 포스트 페이지는 다음과 같이 `generateStaticParams`와 `revalidate` 옵션을 함께 사용하면 **SSG + ISR 하이브리드**가 가능합니다

```tsx
// app/posts/[slug]/page.tsx
export async function generateStaticParams() {
  const posts = getAllPosts();
  return posts.map((post) => ({ slug: post.slug }));
}
```
